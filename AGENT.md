# Efficiency Rules

## Token discipline

* Read before writing. Never rewrite a file you can edit.
* Do not re-read files already in context unless they may have changed.
* Skip files >100KB unless explicitly required.
* Prefer targeted edits over full rewrites.
* When context is large, run /cost to check cache ratio.

## Communication

* No openers, no closers, no filler. Lead with substance.
* Concise output, thorough reasoning.
* One question at a time if clarification is needed.

## Process

* Think before acting. Outline the approach before writing code.
* Test before declaring done.
* Start a new session when switching to an unrelated task.
* Keep solutions simple. Complexity is a bug.

## Priority

User instructions always override this file.
